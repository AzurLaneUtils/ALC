# 版本规范
版本号结构为 `x.y.z-desc.mver`

其中，`-desc`所可能出现的情况如下:

| -desc | 意义 |
|  ---- | ---- |
| -release | 该版本的最后稳定更新 |
| -preview | 预览版本，绝大部分feature已经确定 |
| -develop | 开发版本，feature最新但最不稳定 |

`.mver`是对含有`-desc`版本的描述，代表了`-desc`的第`.mver`个版本。

## 版本变化规则
遵循[语义化版本规范2.0](https://semver.bootcss.com/)
- 当`x`变动时，代表功能、特性或UI界面有重大变化。
- 当`y`变动时，代表添加了新的功能。
- 当`z`变动时，代表了bug的修复。
